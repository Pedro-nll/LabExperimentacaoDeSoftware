KEY INSIGHTS
-----------
1. GraphQL Query Costs
   • Lightweight page scan: ~1 point per 100 PR headers
   • Detail fetch: ~1 point per PR
   • Retries/overhead: ~1 point per repo
   Total: 25-40 points/repo depending on population

2. Rate Limit Behavior
   • Perfectly linear consumption
   • Predictable reset schedule (UTC-0 on hour boundaries)
   • Recommended: Use preflight check before starting

3. Parallelism Strategy
   • 4 workers optimal (tested: 1, 2, 4, 8)
   • Higher concurrency causes rate-limit thrashing
   • 4 workers achieves 26 repos/minute threshold

4. Statistical Sampling
   • 10-sample per repo still valid for most analyses
   • Actually collected 50.9 PRs/repo (higher populations needed more)
   • Deterministic seeding ensures reproducibility

5. Repository Complexity
   • Ultra-large repos (50K+ PRs): 500+ quota each
   • Medium repos (1-10K PRs): 30-40 quota each
   • Small repos (<1K PRs): 10-15 quota each
   • Strategy: Prioritize smaller/medium repos for max coverage
